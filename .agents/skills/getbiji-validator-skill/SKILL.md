# GetBiji Validator Skill

对AI生成的内容进行基于证据的验证，输出可信度评估报告。

## 触发条件

用户意图包含以下关键词时激活：
- 验证、校验、核实、检查
- 这个对吗、准确吗、可信吗
- 有依据吗、来源是什么

## MCP工具绑定

### 必选
- `get-notes-multi-kbs` - 笔记验证
- `knows-mcp` - 医学指南验证

### 医学验证追加
- `pubmed-data-server` - 文献验证

## 验证流程

### Step 1: 主张提取

从待验证内容中提取关键主张：
```
输入: "RMC6236联合埃万妥双抗效果不错，有效率达到60%"

提取主张:
- 主张1: RMC6236可与埃万妥双抗联合使用
- 主张2: 联合用药有效率达到60%
```

### Step 2: 多源验证

对每个主张进行验证：

```
主张: "联合用药有效率达到60%"

验证调用:
1. get-notes-multi-kbs.searchNotes("RMC6236 有效率")
   → 找到 2 条相关笔记
   
2. pubmed-data-server.search("RMC-6236 response rate")
   → 找到 3 篇相关文献
   
3. knows-mcp.searchPapers("RMC6236 efficacy")
   → 找到 1 篇会议报告
```

### Step 3: 支持度评估

```
评估规则:
- 3+来源支持 → 强支持 (✅)
- 1-2来源支持 → 部分支持 (⚠️)
- 0来源支持 → 缺乏支持 (❓)
- 有反例存在 → 有争议 (⚡)
```

### Step 4: 输出报告

## 输出格式

```markdown

### 待验证内容
{原始内容}

### 主张验证结果
```

## 验证权重

| 来源类型 | 验证权重 | 说明 |
|----------|----------|------|
| 医学指南 | 1.0 | 最高权威 |
| PubMed论文 | 0.9 | 同行评审 |
| 个人笔记 | 0.8 | 一手经验 |
| 会议报告 | 0.7 | 未经完整验证 |
| AI搜索 | 0.5 | 需交叉验证 |
| 无来源 | 0.2 | 仅LLM推断 |

## 使用示例

### 输入
```
验证内容: 
"RMC6236是目前最有效的RAS抑制剂，联合埃万妥双抗后
有效率可达60%以上，副作用比传统化疗小很多。"
```

### 执行流程

1. **主张提取**
   - 主张1: RMC6236是最有效的RAS抑制剂
   - 主张2: 联合用药有效率>60%
   - 主张3: 副作用比化疗小

2. **验证调用**
   ```
   get-notes-multi-kbs.searchNotes("RMC6236 有效率 副作用")
   pubmed-data-server.search("RMC-6236 efficacy safety")
   knows-mcp.searchPapers("RAS inhibitor comparison")
   ```

3. **评估输出**
   ```
   主张1: ❓ 缺乏支持 - "最有效"缺乏对比数据
   主张2: ⚠️ 部分支持 - 官方数据，待更多验证
   主张3: ⚡ 有争议 - 不同类型副作用无法直接对比
   ```

### 输出
```markdown
## 验证报告

### 整体可信度: 中

### 建议修改:
1. "最有效" → "新型RAS抑制剂之一"
2. "60%以上" → "初步数据显示约60%（来源：RVMD官网）"
3. "副作用比化疗小很多" → "副作用谱与化疗不同，需个体化评估"
```

## 注意事项

1. **不做主观判断**：只基于证据评估，不加入个人观点
2. **保留争议**：冲突观点都要呈现
3. **标注不确定性**：证据不足时明确说明
4. **可操作建议**：给出具体修改建议
