# GetBiji Writer Skill

将研究报告转化为指定风格的可发布内容。

## 触发条件

用户意图包含以下关键词时激活：
- 专业风格：写论文、报告、文档、技术文档、研究报告
- 科普风格：写公众号、博客、分享、科普文
- 极简风格：写小红书、摘要、速览、笔记

## MCP工具绑定

- `get-notes-multi-kbs` - 引用原始笔记内容（必选）
- `metaso` - 补充写作素材（可选）

## 风格路由

```
用户指令 → 风格选择
"写论文/报告/文档" → 专业风格
"写公众号/博客/分享" → 科普风格  
"写小红书/摘要/速览" → 极简风格
未指定 → 默认科普风格
```

## 专业风格（论文/报告）

### 适用场景
- 学术论文、技术报告、研究综述
- 内部决策文档、方案评审

### 风格要素
| 要素 | 规范 |
|------|------|
| 语气 | 客观严谨、第三人称 |
| 用词 | 专业术语、避免口语 |
| 引用 | 必须标注来源 |
| 数据 | 量化表达、图表优先 |
| 长度 | 2000-5000字 |

### 章节结构

```markdown
# {标题}

## 摘要
{150-300字概括：背景+方法+结论}

## 1. 背景与问题
### 1.1 研究背景
{领域现状、研究意义}
### 1.2 问题定义
{具体问题、研究边界}
### 1.3 研究目的
{期望解决什么}

## 2. 方法与数据
### 2.1 研究方法
{采用的分析框架}
### 2.2 数据来源
| 来源类型 | 数量 | 说明 |
|----------|------|------|
| 个人笔记 | N | 一手实践记录 |
| 医学文献 | N | PubMed检索 |
| 指南共识 | N | 权威指南 |

## 3. 分析与发现
### 3.1 核心发现一
{观点+证据+分析}
### 3.2 核心发现二
{观点+证据+分析}
### 3.3 争议与分歧
{不同来源的冲突点}

## 4. 结论与建议
### 4.1 主要结论
{基于证据的结论}
### 4.2 局限性
{研究局限、证据不足}
### 4.3 行动建议
{可执行的下一步}

## 5. 参考文献
1. [Note#1] {笔记标题}
2. [PubMed] {PMID} - {文献标题}
3. [Guide] {指南名称}

## 附录
{补充数据、详细表格}
```

## 科普风格（公众号/博客）

### 适用场景
- 微信公众号推文、技术博客
- 内部分享、知识传播

### 风格要素
| 要素 | 规范 |
|------|------|
| 语气 | 亲切对话、第二人称 |
| 用词 | 通俗易懂、适度比喻 |
| 故事 | 案例驱动、个人经历 |
| 节奏 | 短段落、多小标题 |
| 长度 | 1500-3000字 |

### 章节结构

```markdown
# {吸引眼球的标题}

> {一句话金句或核心观点}

## 开篇：抛出问题
{用问题或场景引起共鸣，100-200字}

---

## 一、{第一个要点}

{引入}

### 我的经历/案例
{个人笔记中的故事或案例}

### 专家怎么说
{引用权威来源，但用通俗语言解释}

> 💡 **划重点**：{一句话总结}

---

## 二、{第二个要点}

{同样结构：场景→证据→总结}

### 常见误区
{读者可能踩的坑}

### 正确做法
{给出明确建议}

---

## 三、{第三个要点}

{继续分析}

### 数据说话
| 对比项 | 方案A | 方案B |
|--------|-------|-------|
| 优势 | xxx | xxx |
| 风险 | xxx | xxx |

---

## 结语：给你的建议

{基于研究的个人建议，有温度}

### 行动清单
- [ ] 第一步：xxx
- [ ] 第二步：xxx
- [ ] 第三步：xxx

---

📚 **参考资料**
- {来源1}
- {来源2}

---

*{作者署名或引导关注}*
```

## 主题定制模板

### 医学研究主题

**专业风格章节**：
```
1. 背景与临床问题
2. 检索策略与证据来源
3. 药物/方案分析
4. 临床证据评价
5. 推荐与注意事项
6. 参考文献
```

**科普风格章节**：
```
开篇：患者常见困惑
一、这个药/方案是什么？
二、效果怎么样？（数据说话）
三、有什么风险？（副作用坦诚说）
结语：给病友的建议
```

**极简风格**：
```
📌 一句话：xxx
✨ 适用人群：
💊 主要作用：
⚠️ 注意事项：
🔗 详情咨询医生
```

### 技术方案主题

**专业风格章节**：
```
1. 需求背景
2. 方案对比
3. 技术实现
4. 风险评估
5. 结论建议
6. 参考资料
```

**科普风格章节**：
```
开篇：为什么要解决这个问题？
一、几种方案对比
二、我的实践经验
三、踩过的坑
结语：推荐方案
```

## 使用示例

### 输入
```
Researcher输出: {RMC6236联合用药研究报告}
风格: 科普
```

### 执行流程

1. 解析Researcher报告，提取核心发现
2. 查询 `get-notes-multi-kbs` 获取个人案例
3. 按科普风格模板组织内容
4. 添加金句、emoji、行动清单
5. 生成可发布文章

### 输出
```markdown
# RMC6236联合用药：理论很美好，现实怎么样？

> 官方说法很诱人，但临床数据告诉我们更多...

## 开篇：患者的困惑
...
```

## 注意事项

1. **保留引用**：即使极简风格也要标注信息来源
2. **忠于研究**：不能偏离Researcher的结论
3. **风格一致**：全文保持统一语气
4. **图文提示**：需要配图时给出明确提示
